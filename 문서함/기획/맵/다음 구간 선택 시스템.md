# 1. 시스템 개요

본 시스템은 플레이어에게 시각적으로 명확한 선택지를 제공하여 전략적 루트 설계를 유도하는 이지선다형 맵 진행 시스템입니다. 

플레이어의 보상 획득을 트리거로 다음 진행 구간을 그리드 맵 위에 직관적으로 제시하며, 시스템 내부의 가중치 데이터와 고정 등장 룰을 혼합하여 난이도 곡선과 플레이 경험을 통제함.

# 2. 핵심 개념 및 작동 원리

### 노드 (Node / 구간)

플레이어가 진입하게 될 개별 스테이지 단위.

일반 전투, 특수 전투, 상점, 휴식, 보스 등.

### 가중치 (Weight)

해당 노드가 등장할 확률 값.

가중치 총합을 기준으로 난수를 생성해 노드를 추첨하는 가중치 기반 랜덤(Weighted Random) 방식을 사용함.

### 팝업 UI (Popup UI)

현재 위치한 그리드 맵의 다음 타일 위에 노드의 세부 정보(이름, 예상 보상)를 플랫한 형태의 고대비 색상으로 띄워 가시성을 확보하는 방식임.

### 가중치 100 룰

현재 상태에서 가중치가 100 이상으로 누적/설정된 노드는 확률 연산을 무시하고 맵에 확정 등장함.

# 3. 상세 프로세스

## 단계 1 : 생성 트리거 및 초기화

플레이어가 현재 위치한 구간의 클리어 보상을 수령하고 보상 UI를 닫는 순간 다음 구간 생성 로직이 호출됨.

다음 맵 진행을 위해 화면에 띄울 두 개의 선택지 변수 슬롯(`Slot_A`, `Slot_B`)을 빈 값(Null)으로 초기화함.

현재 진행 중인 스테이지의 노드 풀을 불러오고, 각 구간이 현재 가지고 있는 '가중치 데이터'와 ‘최대 등장 횟수'를 로드함.

## 단계 2 : 강제 할당 룰

가중치 기반 랜덤을 돌리기 전, 절대 규칙을 먼저 판별하여 슬롯을 채움.

- 조건 1 : 다음 구간이 ‘**중간 보스**’ 또는 ‘**스테이지 보스**’인가?
    - **Yes** : `Slot_A`에 무조건 ‘휴식’구간을 강제 할당 후 조건 2로 넘어감.
    - **No** : 조건 2로 넘어감.
- 조건 2 : 노드 불 내에 누적/보정 가중치가 `100`이상인 구간이 존재하는가?
    - **Yes** : `Slot_A`또는 `Slot_B`에 해당 구간을 확정적으로 할당함.
    - **No** : 단계 3으로 넘어감.

## 단계 3 : 가중치 추첨

강제 할당 후에도 비어있는 슬롯이 있다면 가중치 랜덤 추첨을 진행.

`Slot_A`가 비어있을 경우, 현재 노드 풀의 가중치 총합 내에서 난수를 돌려 구간 하나를 뽑아 `Slot_A`에 할당함.

`Slot_A`에 할당된 구간이 비전투 구간이면 `Slot_A`에 있는 비전투 구간의 가중치를 일시적으로 `0`으로 만듬.

`Slot_B`가 비어있을 경우, 가중치 총합 내에서 다시 난수를 돌려 남은 구간 하나를 뽑아 `Slot_B`에 할당함.

## 단계 4 : UI 출력

결정된 두 슬롯의 데이터 기반으로 UI에 표기할 [다음 구간 이름]과 [기대 보상] 텍스트를 매핑함.

구간 속성에 맞춰 팝업 UI의 배경 컬러를 지정함.

[그리드 맵 시각화 및 컬러 배경](https://app.notion.com/p/37b74a129df0809bb79efffbc2e9d4af?pvs=21) 참고.

그리드 맵 상의 다음 진행 타일 2곳 위에 완성된 팝업 UI를 노출시킴. 

## 단계 5 : 유저 인터랙션

플레이어가 2개의 팝업 중 하나를 클릭하여 진입함.

선택된 구간은 정해진 규칙에 따라 ‘최대 등장 횟수’를 1차감하고 초기 가중치로 돌아감.

선택 받지 못한 특정 구간은 다음 추첨을 위해 보정 가중치가 누적됨.

비전투 구간과 특수 전투 구간은 해당 기획서에 표기된 규칙을 따름.

# 4. 세부 규칙

### 가중치 기반 추첨 규칙

일반 전투, 특수 전투, 상점, 휴식은 각각 독립적인 가중치 값을 가짐.

구간 리스트의 총 가중치 합 안에서 난수를 돌려 선택지를 뽑음.

조건 부합 등으로 특정 노드의 가중치가 `100`이상이 되면 무조건 해당 노드가 선택지 슬롯 중 하나에 확정 배치됨.

### 그리드 맵 시각화 및 컬러 배경

플레이어의 즉각적인 인지를 위해 팝업 UI의 배경색은 굵은 외곽선과 단색 혀애의 디자인을 따르며 아래의 룰을 적용함.

- 전투 구간(일반/특수) : 주황색
- 비전투 구간(상점/휴식) : 파란색
- 보스전(중간/스테이지) : 빨간색

### 비전투 구간 중복 등장 방지 룰

- 2개의 선택지가 모두 동일한 비전투 구간으로 채워지는 것을 금지함.
    - 예: 불가[상점-상점] / [휴식-휴식], 가능[상점-전투]
- 첫번째 선태지로 비전투 구간이 추첨될 경우, 두 번째 선택지 추첨 풀에서 해당 노드의 가중치는 일시적으로 0으로 계산됨.

### 보스전 직전 휴식 확정 등장 룰

다음 페이지가 중간 보스 또는 스테이지 보스일 경우, 현재 생성되는 2개의 선택지 중 1개는 무조건 ‘**휴식**’ 구간으로 고정됨.

이때 등장하는 강제 ‘휴식’은 기존 휴식 노드가 가진 ‘최대 등장 횟수’를 **차감하지 않음**.

‘최대 등장 횟수’를 모두 소모한 상태라도 **강제로 등장**하며, 등장 시 발생하는 기존의 보정 가중치 초기화 규칙, 쿨타임 규칙 등도 **일절 적용받지 않음**.

# 5. 데이터 구조

#### 구간별 데이터 테이블

**Node_Id** : 노드 고유 번호

**Node_Type** : 구간의 분류, 타입 종류[일반 전투, 특수 전투, 상점, 휴식, 중간보스, 스테이지 보스 등]

**UI_Color** : 해당 타입에 맞는 색상 배치

**Base_Weight** : 기본 가중치.

**Max_Limit** : 최대 등장 수량

# 6. 예외 처리 방안

### 가중치 100이상 노드가 3개 이상일 경우

선택지는 2개인데 확정 등장 조건(100 이상)을 만족하는 노드가 초과 발생할 수 있음.

노드 타입별 우선순위 값을 참조하여 가장 높은 2개만 확정 등장시키고, 나머지는 다음 타일 생성 시로 이월함

**우선순위: 1순위 강제 룰, 2순위 특수 전투, 3순위 비전투, 4순위 일반 전투**

### 보스전 직전 룰과 가중치 100 룰의 충돌

시스템 상 '보스전 이전 휴식 확정 룰'을 최상위 조건문으로 배치하여, 다른 어떤 가중치 보정보다 우선하여 1개 슬롯을 차지하도록 설계함.
