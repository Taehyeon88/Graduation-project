# 대미지 연산 시스템

---

최신 업데이트 : 2026.04.04  
버전 : 0.0.2 (항목별 기획 의도 작성)  
---

## 1\. 개요

게임 내 발생하는 모든 대미지의 산출 및 적용 방식을 정의합니다. 버프, 디버프, 카드 효과 등 다양한 변수가 충돌 없이 적용되도록 연산의 우선순위와 소수점 처리 방식을 엄격하게 통제합니다.  
또한 카드, 버프/디버프, 위치 선정(충돌) 등 다양한 변수가 교차하는 전략 게임이므로, 룰이 복잡할수록 기반 연산식은 ‘견고하고 직관적’이어야 합니다. 무한한 대미지 인플레이션을 막고 유저가 결과를 어느 정도 예측할 수 있도록 통제하는 것이 핵심입니다.  
기획서에 명시되지 않은 신규 형태의 버프/디버프 효과가 추가되더라도, 플로그래머는 반드시 이 문서의 연산 파이프라인 위에 얹혀야만 버그(연산 충돌)가 발생하지 않음을 인지해야합니다.

## 2\. 대미지 타입 분류

시스템 내의 피해는 다음 3가지 타입으로 분류되며, 각각 별도의 연산 파이프라인을 가집니다.

1. 스킬 피해 : 카드를 사용하여 직접 가하는 피해 (증감 계수 적용 O)  
2. 고정/충돌 피해 : 넉백, 충돌, 특정 버프에 의해 발생하는 고정 피해 (증감 계수 적용 X)  
3. 상태이상 피해 : \[출혈\], \[중독\] 등에 의해 턴 시작 시 발생하는 최대 체력 비례 피해 (전용 증감 계수 적용)

	\[기획 의도\]

* 방어력이 너무 높은 적에게는 \[틱 피해\]를, 위치가 취약한 적에게는 \[고정/충돌 피해\]를 유도하여 단순 수치 싸움이 아닌 ‘기믹 싸움’을 제공하기 위함입니다.  
* \[개발 주의\] 각 피해 타입별로 ‘증감 계수(버프/디버프)’가 적용되는지 여부가 다르므로, 대미지 타입값에 따라 타는 로직을 완전히 분리하여 구현해야 합니다.

## 3\. 핵심 산출 공식 (카드 피해)

카드 사용 시 발생하는 스킬 피해는 아래의 단일 공식을 표준으로 사용합니다.

공식 : **최종 피해량 \= Ceil(Max(0, 기본 피해량 \+ 고정 수치 증감) x 공격자 계수 ) x 방어자계수**

(※ Ceil() : 괄호 안의 최종 결과값에서 발생하는 소수점을 무조건 올림 처리하여 정수로 반환)  
(※ Max(0, A) : 괄호 안 값 중 더 큰 값을 선택(반환)처리하여 최소 0으로 처리)

### 3.1. 산출 공식 세부 설명 및 의도

* **고정 수치(+N) 우선, 퍼센트(xN%) 나중** : 초반 장비/버프의 효용성을 극대화하기 위함입니다. 고정 수치가 먼저 더해져야 이후 곱해지는 \[집중\], \[표적\] 등의 시너지 가치가 기하급수적으로 높아집니다.  
* **Max(0, 값) 함수의 강제 적용** : 방어력이 너무 높거나 디버프가 과도하게 쌓여 최종 대미지가 음수 (-)가 될 경우, 공격했는데 적 체력이 회복되는 치명적인 버그를 원천 차단하기 위해 0미만으로 떨어지지 않게 강제합니다.  
* **소수점 무조건 올림(Ceil) 처리** : 대미지가 0.1이라도 발생했다면 최소 1의 피해를 보장하여 유저의 액션(카드 사용)이 무의미해지는 박탈감을 방지합니다. 프로그래머는 임의로 반올림, 내림, 캐스팅을 사용해선 안됩니다.

### 3.2. 기본 피해량

* 고정형 : 카드에 명시된 고정 수치  
* 동적형 : 특정 스탯에 비례하는 수치. 연산 중 소수점 발생 시 올림(Ceil) 처리  
  * 예시: \[육중한 압박\] \= Ceil(현재 방어도 \* 1.5)  
* 조건부 추가 피해 합산 : 특정 조건을 만족하여 발생하는 추가 피해는 기본 피해량에 합산되어 한 번의 타격으로 계산됩니다.  
  * 예시 : 기본 피해 4, 조건 달성 시 추가 피해 8인 카드에서 조건 달성 \-\> 기본 피해량 을 12(4+8)로 취급하여 단일 연산

\[기획 의도\]

* 기본 피해량을 고정형으로 두는 이유는 유저가 카드의 ‘최소 밸류’를 직관적으로 파악하게 하기 위함입니다. 카드 레벨업이나 강화 시스템이 추가될 경우, 이 기본 피해량 자체가 상승하는 구조를 가져가 성장 체감을 뚜렷하게 합니다.

### 3.3. 고정 수치 증감

비율(%) 기반의 계수가 곱해지기 전에 기본 피해량에 직접 더해지거나 뼤지는 고정 수치입니다. (기본값 0\)

* 적용 대상:  
  * 증가(+) : 장비/유물에 의한 기본 공격력 \+N 증가, 특정 능력 카드 사용으로 인한 스킬 피해량 \+N 증가 등  
  * 감소(-) : 특정 적의 방어 기믹에 의한 받는 피해량 \-N 감소, 디버프에 의한 주는 피해량 \-N 감소 등  
* 목적 : 계수 연산 전에 적용되는 \[집중\], \[혼란\] 등의 퍼센트 버프/디버프 효율에 직접적인 영향을 줍니다.  
* 예외 처리 (0 미만 방지) : 기본 피해량에서 고정 수치를 뺀 결과값이 0 미만(음수)이 될 경우, 무조건 0으로 보정한 뒤 다음 퍼센트 계수 연산으로 넘어가야 합니다.

	

### 3.4. 공격자 계수

공격자가 가진 딜링 관련 버프/디버프의 비율 총합입니다. (기본값 1.0)

* 연산 방식 : 합연산  
* 수식 : Max(0, 1.0 \+ 공격자 버프 총합 \- 공격자 디버프 총합)  
* 해당 요소:  
  * \[집중\] : \+0.25 (25 증가)  
  * \[혼란\] : \-0.50 (50% 감소)  
  * \[견제 사격 : 다음인접카드 사용시\] : \+0.50 (50% 증가)

\[기획 의도\]

* **계수의 합연산 처리** : 버프가 여러 개 겹칠 때 곱연산이 아닌 합연산을 사용하는 이유는 특정 버프 조합 시 대미지가 천문학적으로 튀어오르는 오버 밸런스 현상을 방지하기 위함입니다.  
* 디버프가 아무리 강력하게 걸리더라도 Max(0) 처리를 통해 공격자 계수가 음수로 떨어져 대미지가 역전되는 현상을 코드로 완벽히 차단해야 합니다.

### 3.5. 방어자 계수

방어자가 가진 피격 관련 버프/디버프으 비율 총합입니다. (기본값 1.0)

* 연산 방식 : 합연산  
* 수식 : Max(0, 1.0 \+ 방어자 디버프 총합 \- 방어자 버프 총합)  
* 해당 요소:  
  * \[표적\] : \+0.50(50% 증가)

\[기획 의도\]

* **방어자 우위의 밸런스 의도** : 몬스터의 방어 효과 (받는 피해 감소 등)가 플레이어의 공격 버프를 상회하여 대미지가 크게 깍이거나 0에 가깝게 나오는 것은 기획적으로 의도된 정상적인 상황입니다.  
* **우회 공략의 유도** : 해당 몬스터의 패턴이 방어력 증가라면, 플레이어는 버프를 발라 억지로 뚫어내는 대신 아래와 같은 유기적인 대처를 하도록 유도합니다.  
1. **디버프 상쇄** : 방어력 상승효과를 \[표적\]같은 디버프를 걸어 연산식 내에서 상쇄시킵니다.  
2. **타입 변경** : 카드 피해가 막히면 방어자 계수를 무시하는 \[상태이상 틱 피해\]나 \[충돌/벽쿵 고정 피해\]를 활용해야 유리합니다.  
3. **그리드를 활용한 턴 셋업** : 슬레이어 더 스파이어 등 타 게임과 달리 Tlop은 그리드 이동이 가능합니다. 몬스터가 방어 버프를 획득한 턴에는 무히하게 공격하지 않고, 그리드를 이용해 적의 공격 범위에서 벗어나거나 다음 턴을 위한 셋업(영역 배치, 버프 획득 등)을 진행하는 템포 조절 턴으로 활용하도록 설계되었습니다.  
* 즉, 공격자 산출이 완전히 끝난 결괏값에 방어자 계수를 독릭적으로 곱하는 이 구조는, 유저에게 단순 공격이 아닌 ‘공간과 디버프를 활용한 전술적 선택’을 강제하기 위한 핵심 장치입니다.

## 4\. 특수 피해 연산 산출 공식

일반적인 공격/방어 계수를 타지 않는 예외적인 대미지 산출식입니다.

### 4.1. 턴 시작 상태이상 피해

* 대상 상태이상 : \[출혈\], \[중독\]  
* 공식 : 틱 피해량 \= Ceil(타겟 최대 체력 \* 0.03 \* 특수 증폭 계수)  
* 특수 증폭 계수:  
  * 기본 값 : 1.0  
  * 타겟에게 \[악화\] 디버프 존재 시 2.0 (2배 증폭)

\[기획 의도\]

* 체력 비례 피해 의도 : 방어력이 극단적으로 높거나 체력이 기형적으로 많은 보스/탱커 몬스터를 카운터 치기 위한 전술적 우회로입니다.  
* 무조건적인 상태이상 남용을 막기 위한 기본 배율은 낮게(3%) 설정하되, \[악화\]와 같은 특정 디버프를 연계했을 때 리턴(2배)를 주어 유저에게 ‘콤보’사용을 유도합니다.

### 4.2. 고정 및 충돌 피해

* 공식 : 최종 피해량 \= 명시된 고정 수치  
* 적용 대상:  
  * 넉백 충돌(벽쿵) 피해  
  * \[공성 전차\] 버프에 의한 충돌 시 추가 피해

\[기획 의도\]

* 위치 기반 전술 강조 : 적의 방어력이나 회피율이 아무리 높아도, 넉백을 통해 벽이나 다른 유닛에 충돌시키면 무조건적인 ‘고정 피해’를 줄 수 있게 하여 공간 활용의 재미를 극대화합니다.  
* \[개발 주의\] 이 대미지는 일반 스킬의 공격/방어 계수를 절대 참조하지 않습니다. 예외 처리 없이 명시된 수치 그대로 적용되도록 구현바랍니다.

## 5\. 핵심 예외 처리 및 주의사항

### 5.1. 단일 타격(합산) vs 개별 타격(분리) 판정 기준

카드 효과로 발생하는 피해의 성질에 따라 연산 및 타격 횟수가 달라집니다.

1. 조건부 추가 피해 (단일 타격 판정)  
   * 설명 : 단순히 타겟의 상태나 위치 조건을 만족했을 때 피해량이 늘어나는 경우입니다.  
   * 처리 방식 : 기본 피해량과 추가 피해량을 합산하여 한 번의 타격(1 Hit)으로 연산합니다.  
   * 예시 : 기본 4피해를 주고, 타겟이 \[표적\] 상태라면 8의 추가 피해를 줌 \-\> 기본 피해량을 12로 취급하여 Ceil(Max(0, 12 \+ 고정수치증감) \* 공격 계수 \* 방어계수) 공식에 대입  
2. 넉백 충돌 피해 (개별 타격 판정)  
   * 설명 : 카드 사용으로 대미지를 입힌 후 , 타겟이 밀려나서 (넉백) 벽이나 다른 개체와 부딪혔을 때나 공격을 여러번(다수)일 경우의 피해입니다.  
   * 처리 방식 : 스킬에 의한 기본 타격과, 충돌 시 발생하는 물리적 피해 또는 여러 여러번(다수)일 경우의 피해를 독립된 2번(2 Hit) 또는 N번(N Hit) 타격으로 분리하여 연산합니다.  
     * 예시: 넉백에 의한 피격 시퀸스:  
       1. 스킬 기본 피해 연산 및 적용 (1타)  
       2. 넉백 이동 처리  
       3. 이동 종료 지점에서 충돌 발생 시, 충돌 피해 연산 및 적용 (2타, 독립적 계산)  
     * 예시: N번 피격 시퀸스:  
       1. 스킬 기본 피해 연산 및 적용 (1타)  
       2. N번 1번 반복

\[기획 의도\]

* **조건부 대미지를 합산 이유** : 만약 타격을 분리할 경우, “적 타격 시 마다 발동”류의 유물/버프 효과가 1번의 스킬 사용에 여러번 터지게 되어 밸런스가 심각하게 무너집니다. 묵직한 한 방의 형태로 합산해야합니다.  
* 넉백/충돌을 분리하는 이유 : 시각적 애니메이션 시퀸스(때림 \-\> 날아감 \-\> 벽에 부딪힘)와 실제 대미지 발생 시점을 일치시켜 타격감과 물리적 설드력을 주기 위함입니다.

## 6\. 소수점 처리 원칙 (매우 중요)

* 게임 내의 모든 대미지 연상에서 소수점이 발생할 경우, 과정 연산(동적 피해량 산출 등)과 결과 연산 (최종 스킬/틱 피해량 산출)을 불문하고 언제나 올림(Ceil)을 사용합니다.  
* 이를 통해 최종 적용 수치가 무조건 정수로 떨어지도록 보장합니다.
