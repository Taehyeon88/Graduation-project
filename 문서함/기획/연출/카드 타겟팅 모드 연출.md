# 카드 타겟팅 모드 연출

---

최신 업데이트 : 2026.04.11  
버전 : 5.0.0.2 (횡베기, 기술카드 (방어, 버프, 영역) 4.5, 4.6, 4.7 항목 추가)

---

## 1\. 개요

플레이어가 카드를 ‘산택된 상태’로 전환했을 때 진입하는 ‘타겟팅 모드’의 작동 방식과, 마우스 오버 시 나타나는 타일 UI피드백, 그리고 카드 타입별 타겟팅 규칙을 정의합니다.

## 2\. 기본 조작 흐름

1. **상태 진입** : 플레이어가 핸드에서 카드를 클릭/드래그하여 \[선택된 상태\]가 됨  
2. **범위 출력** : 그리드 맵 상에 해당 카드의 \[공격 가능 범위\]가 하이라이트 되어 표시됨  
3. **마우스 호버** : 플레이어의 마우스 포인트가 \[공격 가능 범위\] 내의 특정 타일에 진입함  
4. **타겟팅 출력** : 마우스가 위치한 곳을 기준으로 카드의 실제 \[공격 범위\]기 타일 위에 표시됨  
5. **유효성 검사** : 표시된 \[공격 범위\] 넹\[ 몬스터 존재 여부 및 카드 효과 적용 가능 여부를 시스템이 판별함  
6. **결과 피드백** : 판별 결과에 따라 \[공격 표시(사용 가능)\] 또는 \[공격 불가 표시(사용불가)\]로 타일에 문양이 전환됨

## 3\. UI/UX 사양 및 피드백 규칙

### 3.1. 타일 표시 타입

\[타일 시각 효과 및 인터랙션 기획서\]의 시각 요소 정의에 따라, 타겟팅 결과에 대한 시각적 피드백은 타일의 \[문양\] 요소를 통해 표현됩니다.  
‘타겟팅 모드’에서 그리드 맵에 표시된 ‘공격 가능 범위’에 마우스 호버 시 대상 타일에 2가지 형태의 시각적 피드백이 제공됩니다.

* **타입 A : 공격 표시**  
  * **조건** : 타겟팅한 공격 범위 내에 몬스터가 존재하거나 조건이 충족된 경우  
  * **상태** : 카드 사용(시전) 가능  
  * **시각 요소\[문양\]** : 타겟 위치에 나타나는 \[주황색 문양 테두리\].  
  * ~~추가 인터랙션 : 마우스 커서가 \[칼 모양\]으로 변경되며, 몬스터 머리 위에 \[예상 대미지 수치\]가 표시됨~~  
* **타입 B : 공격 불가 표시**  
  * **조건** : 타겟팅한 공간에 몬스터가 없거나, 해당 타겟에 카드 효과를 부여할 수 없는 경우  
  * **상태** : 카드 사용(시전) 불가 (클릭 및 드롭 시 발동 무시)  
  * **시각 요소\[문양\]** : 타일 중앙에 명확한 ‘X자 모양’ 문양 표시  
* **타입 C : \[본인 사용 표시\]**  
  * **조건** : 방어, 버프 등 자신에게 사용하는 기술 카드를 시전 가능한 상황일 경우  
  * **상태** : 카드 사용(시전) 가능  
  * **시각 요소\[문양\]** : 타겟(시전자 본인) 위치에 나타나는 \[파란색 문양 테두리\].

### 3.2. 시스템 경고 문구

* **발생 조건** : \[공격 불가 표시 (X자 문양)\]가 뜬 상태에서 플레이어가 카드 사용을 시도(클릭 또는 드롭 또는 단축키 사용)할 경우  
* **출력 위치** : 화면 상단 중앙에서 살짝 아래  
* **출력 방식** : 플로팅 텍스트 형태로 나타났다가 서서히 사라짐  
* **출력 텍스트** : “타겟팅할 대상이 없습니다.”

## 4\. 카드 타입별 타겟팅 규칙

카드의 공격/효과 범위 형태에 따라 몬스터 존재 여부를 판별하는 기준이 다르게 적용됩니다.

### 4.1. 단일 타겟 카드 (인접\_단일, 투사\_단일, 디버프)

* **공격 범위** : 1칸 (단일 타일)  
* **판별 규칙**:  
  * 해당 1칸 타일에 몬스터가 있음 \-\> \[공격 표시\]  
  * 해당 1칸에 몬스터가 없음 \-\> \[공격 불가 표시\]  
* **예외 처리** : \[악화\]카드는 본 규칙에서 제외되며, 타겟의 디버프 상태를 추가로 검사합니다.  
  * 상세 내용은 4.8. 특수 조건 카드 항목 참조

### 4.2. 관통 타겟 카드 (인접\_관통, 투사\_관통)

* **공격 범위** : 직선 2칸  
* **판별 규칙 (부분 교집합 허용)**:  
  * 해당 2칸 타일 중 몬스터가 1마리라도 있음 \-\> \[공격 표시\] (두 칸 모두 정산 타겟팅 타일로 표시)  
  * 해당 2칸 타일 중 몬스터가 1마리도 없음 \-\> \[공격 불가 표시\] (두 칸 모두 \[공격 불가 표시\])

### 4.3. 돌진 공격 카드 (인접\_돌진)

해당 카드는 타겟에 대한 공격 여부와 무관하게 ‘이동’자체가 주 목적인 카드입니다.

* **판별 규칙 (몬스터 유무 무관)**:  
  * 타겟팅(도착) 경로에 몬스터가 있든 없든 항상 카드를 사용할 수 있습니다. (정산 타겟팅 UI표시)  
  * 즉, 몬스터가 한마리도 없는 빈 공간을 지정하여 단순 이동만 진행하더 라도 \[공격 불가 표시(X자 문양)\]가 나타나지 않습니다.

### 4.4. 폭발 타겟 카드

해당 카드는 메인 타겟과 주변 스플래시 범위를 모두 묶어서 유효성을 검사합니다.

* **공격 범위** : 총 5칸 (메인 타겟 타일 1칸 \+ 주변 X자 범위 스플래시 타일 4칸)  
* **판별 규칙**:  
  * 5칸의 범위 중 단 1마리라도 몬스터가 존재할 경우 (메인 공격 타일이 비어있더라도 스플래시 범위에 몬스터가 있다면) \-\> 5칸 전체에 \[공격 표시\]  
  * 5칸의 범위 내에 몬스터가 1마리라도 없을 경우 \-\> 5칸 전체에 \[공격 불가 표시\]

### 4.5. 횡베기 공격 카드

* **공격 범위** : 총 3칸  
* **판별 규칙**:  
  * 해당 3칸 타일 중 몬스터가 1마리라도 있음 \-\> \[공격 표시\] (두 칸 모두 정산 타겟팅 타일로 표시)  
  * 해당 3칸 타일 중 몬스터가 1마리도 없음 \-\> \[공격 불가 표시\] (두 칸 모두 \[공격 불가 표시\])

### 4.6. 기술 카드 (방어, 버프)

해당 카드는 적이 아닌 플레이어(본인)를 대상을 시전하는 카드입니다.

* 판별 규칙:  
  * 카드를 사용 가능한 상황에서 플레이어(본인)에게 타겟팅 될 경우 \-\> \[본인 사용 표시\]

### 4.7. 영역 기술 카드 (필드, 오브젝트)

해당 카드는 몬스터 타격이 아닌 필드 설치가 주 목적인 카드들입니다. 두 계열의 장애물 인식 기준이 다릅니다.

* 판별 규칙 (몬스터 유무 무관):  
  * **필드 계열** : 벽 등의 맵 장애물을 제외한 모든 타일을 유효 범위로 출력하며, 해당 범위에 호버 시 \-\> \[공격 표시\] 출력  
  * **오브젝트 계열** : 벽 뿐만 아니라 개체(몬스터)가 위치한 타일도 장애물로 인식합니다. 즉, 애초에 몬스터가 있는 자리는 ‘공격 가능 범위’ 하이라이트에서 제외되며, 출력된 유효 공간에 호버 시 \-\> \[공격 표시\] 출력

### 4.8. 특수 조건 카드 (악화)

해당 카드는 단순 몬스터의 위치뿐만 아니라, 몬스터가 현재 가지고 있는 상태이상(디버프)를 조건으로 유효성을 검사합니다.

* **공격 범위** : 1칸 (단일)  
* **판별 규칙**:  
  * 타겟팅한 타일에 몬스터가 존재하며, 해당 몬스터에게 \[틱피해\] 계열의 디버프가 있을 경우 \-\> \[공격 표시\]  
  * 타겟팅한 타일에 몬스터가 존재하지만, 해당 몬스터에게 \[틱피해\] 계열 디버프가 없을 경우 \-\> \[공격 불가 표시\] (조건 불충족)  
  * 타겟팅한 타일에 몬스터가 없을 경우 \-\> \[공격 불가 표시\]
