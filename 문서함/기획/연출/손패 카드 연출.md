# 손패 카드 연출

---

최신 업데이트 : 2026.04.16  
버전 : 1.0.0.5 (턴 시작 시 카드 잠금 연출 제거)

## 1\. 개요

전투 시스템 중 핵심인 카드 사용 및 턴 진행 방식에 대한 UI/UX 및 시스템 기획서입니다.  
플레이어는 그리드 기반의 맵에서 턴제로 전투를 진행하며, 손패에 쥐어진 카드를 조작(마우스 클릭/드래그, 단축키)하여 적을 공격하거나 자신을 보조합니다. 턴 시작 시 강제 이동 조건 등 전략적인 행동 제약이 포함되어 있으며, 플레이어가 이를 직관적으로 인지할 수 있도록 하는 피드백 연출이 포함되어 있습니다.

## 2\. 카드 기본 인터랙션 (마우스 및 단축키)

플레이어가 손패(Hand)에 있는 카드와 상호작용할 때의 기본 조작 방식입니다.

* **마우스 호버**  
  * **조건** : 손패에 있는 카드 위에 마우스 포인터를 올림  
  * **피드백**:  
    * **시각 피드백**

1. **포커싱** : 해당 카드가 손패 배열에서 살짝 위로 올라오며(팝업) 동시에 카드의 크기가 원래보다 살짝 커짐(Scale-Up 약 1.16% / 16% 확대). 카드의 상세 정보를 온전히 보여줌. 이때, 겹쳐 있는 주변 카드(특히 우측 카드)에 의해 내용이 가려지지 않도록 해당 카드가 화면의 가장 앞쪽(최상단 렌더링)으로 배치되어야 함

2. **인접 카드 밀림 연출** : 호버된 카드가 커짐에 따라, 해당 카드의 양옆에 위치한 카드들이 공간을 내어주듯 자연스럽게 양옆으로 밀려나는 애니메이션이 적용

3. **카드 간 호버 전환** : 마우스 포인터가 호버된 카드 표면에서 바로 옆에 위치한 카드로 넘어갈 경우, 새로 마우스가 닿은 카드가 즉시 호버 피드백(위로 팝업 \+ 1.16배 확대 \+ 맨 앞 렌더링)을 받으며 주변 카드를 밀어냄. 동시에, 기존에 호버되었던 카드는 즉시 원래 크기(1.0)와 기본 Y축 높이로 복귀하며 새로 커진 옆 카드에 의해 자연스럽게 옆으로 밀려나는 위치에 자리잡게 됨

4. **호버 취소** : 마우스 포인터가 손패 영역을 완전히 벗어나 호버가 취소되며느 커졌던 카드는 원래 크기와 위치로 복귀하며, 양옆으로 밀려났던 카드들도 부드럽게 제자리로 돌아감  
   * **청각 피드백**  
     * **사운드** : SND\_UI\_Card\_Hover 재생

* **단축키 (키보드 1\~0)**  
  * **조건** : 손패의 맨 좌측 카드부터 우측으로 순서대로 숫자키 1 부터 0까지 할당됨  
  * **피드백**:  
    * **UI** : 단축키를 누르면 해당 카드가 즉시 ‘선택된 상태 (Active)’로 전환됨  
    * **사운드** : SND\_UI\_Card\_Select 재생

* **마우스 조작 (클릭 및 드래그)**  
  * 카드를 마우스로 조작(클릭 또는 드래그)하여 ‘선택된 상태(Active)’로 만들거나 카드를 즉시 발동시킬 수 있습니다. (상세한 카드 타입별 작동 방식은 ‘4. 카드 사용 로직’에서 후술)  
  * **피드백**:  
    * **UI** : 카드가 호버 상태보다 좀 더 높이 위로 올라가며, 커진 크기(약 1.16% 확대)를 유지한 채 ‘선택된 상태 (Active)’로 전환됨  
    * **사운드** : SND\_UI\_Card\_Select 재생

## 3\. 카드 타입별 타겟팅 및 UI 표시

카드가 선택된 상태(Active)가 되어 타겟팅 모드에 돌입하면, 카드의 목적에 따라 그리드 상에 표시되는 UI가 두 가지 타입으로 나뉩니다.

**타입 A : 대상 지정형 (공격, 영역, 이동, 디버프 카드)**

* **목적** : 적에게 피해/디버프를 주거나 플레이어 캐릭터가 특정 위치로 이동하는 등 ‘외부 타일’에 간섭하는 목적의 카드

* **UI 피드백** : 그리드 맵 상에 ‘사용 가능한 범위(공격 가능 범위, 이동 가능 범위)’가 하이라이트 되어 표시됨

**타입 B: 자가 적용형 (버프, 능력, 방어 카드)**

* **목적** : 플레이어 캐릭터 자신에게 이로운 효과를 부여하는 목적의 카드

* **UI 피드백** : 맵 상에 넓은 범위를 표시하지 않고, 현재 플레이어 캐릭터가 위치한 타일에만 카드 효과가 적용됨을 알리는 하이라이트가 표시됨

## 4\. 카드 사용(소모) 및 처리 로직

선택된 카드를 발동하고 소모하는 방식은 유저의 편의성(조작 피로도 감소)을 고려하여 카드 타입별로 다르게 작동합니다.

### 4.1. 타입 A (대상 지정형) 사용 방식

* **선택 조작** : 카드를 마우스로 클릭하거나 드래그할 때, 또는 단축키 입력 시 타겟팅 Ui가 표시되는 ‘타겟팅 모드’로 진입  
* **발동/소모 조작** : 타겟팅 모드 상태에서 하이라이트된 공격/이동 가능 그리드 타일 중 하나를 마우스로 클릭하면 카드가 발동되며 소모

### 4.2. 타입 B (자가 적용형) 사용 방식

플레이어 자신에게 사용하는 카드이므로, 불필요한 타일 클릭 과정을 생략하고 즉시 발동할 수 있는 퀵 캐스팅을 지원합니다.

* **선택 조작** : 카드를 마우스로 클릭하거나 단축키(1회) 입력 시 타겟팅 모드로 진입 (플레이어 위치 타일 하이라이트)  
* **발동/소모 조작 (즉시 발동)**  
  * **마우스** : 손패에서 기본 상태에서 카드를 곧바로 위로 드래그하면, 타겟팅 모드 연출을 거쳐 바로 카드가 발동 및 소모  
  * **단축키** : 타겟팅 모드 상태에서 해당 카드를 단축키를 한 번 더 입력(더블 탭)하면 즉시 카드가 발동 및 소모

### 4.3. 카드 소모 후 위치

카드가 발동되어 소모될 때, 제자리에서 단순히 사라지지 않고 카드의 속성에 따라 각 목적지 UI 위치를 향해 날아가며 점차 크기가 작아지는 트위닝 애니메이션 연출이 재생됩니다.

* **일반 카드** : 사용 후 화면 내  \[버린 카드 더미\] UI 위치를 향해 날아가며 크기가 줄어들고 들어가는 연출 진행  
* **능력 카드** : 사용 후 화면 내 \[소멸 카드 더미\] UI 위치를 향해 날아가며 크기가 줄어 들고 소멸되어 해당 전투 중 재사용 불가

## 5\. 상태 이상에 의한 카드 사용 불가 처리

전투 중 몬스터의 패턴이나 카드 디메리트 효과로 인해 특정 타입의 카드를 사용할 수 없게 되는 상황의 처리 방식입니다. (예 : ‘고립’ 상태이상 시 ‘이동’ 카드 사용 불가)

* **시각적 연출** : 턴 시작 시의 잠금 처리와 동일하게, 사용 불가 상태가 된 카드만 회색 톤으로 변환하여 직관적으로 알려줌  
* **인터랙션 피드백 (턴 시작 잠금과의 차이점)**  
  * 마우스 호버 시 : 카드가 살짝 위로 올라와 확인은 가능함. 단, 턴 시작 때와 달리 마우스 포인터 옆에 별도의 안내 문자(툴팁)는 뜨지 않음  
  * 조작(클릭/단축키) 시도 시 : 카드가 선택되지 않으며 타겟팅 모드로 진입하지 못함. (단순히 비활성화 된 상태임을 어필하며 팝업 경고 메시지 없음)  
  * 사운드 : SND\_SYS\_Warn\_Err 재생
